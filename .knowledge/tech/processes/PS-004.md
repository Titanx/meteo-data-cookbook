## 背景

GOES（Geostationary Operational Environmental Satellite）是美国 NOAA 运行的静止气象卫星系列，覆盖美洲及周边海域（西半球），提供 5-15 分钟间隔的高频观测，16 个 ABI 波段（可见光、近红外、红外），是美洲区域最常用的静止卫星数据源。

与 Himawari（覆盖东半球）类似，GOES 数据通过 AWS Open Datasets 免费提供，匿名访问无需凭证。本条目基于 2026-07-20 的实际测试验证，已成功下载实时数据并生成真彩色图。

### 目录结构

```
noaa-goes19/
├── ABI-L1b-RadF/
│   └── {YYYY}/                               # 年份
│       └── {DDD}/                            # 儒略日（001-366）
│           └── {HH}/                         # 小时（UTC）
│               └── OR_ABI-L1b-RadF-M6C{XX}_G19_s{YYYY}{DDD}{HH}{MM}{SS}...
```

**与 Himawari 的关键差异**：
- GOES 目录用**儒略日**（day of year），Himawari 用月/日
- GOES 文件为 **NetCDF4 (.nc)**，Himawari 为 HSD (.DAT.bz2)
- GOES 每次扫描每通道是**单文件**，Himawari 分 10 个纬度分段

### 文件命名规则

**L1b 文件**：`OR_ABI-L1b-RadF-M6C{XX}_G{NN}_s{YYYY}{DDD}{HHMMSS}{SSS}_e{...}_c{...}.nc`

| 字段 | 含义 | 示例 |
|------|------|------|
| `OR` | Operational Real-time | `OR` |
| `ABI-L1b-RadF` | ABI L1b 全圆盘辐射 | `ABI-L1b-RadF` |
| `M6C{XX}` | Mode 6, Channel XX (01-16) | `M6C01` |
| `G{NN}` | 卫星编号 | `G19` |
| `s{YYYY}{DDD}{HHMMSS}` | 扫描开始时间 | `s2026201013019` |

### Python 下载示例

```python
import boto3
from botocore import UNSIGNED
from botocore.config import Config
from datetime import datetime, timezone, timedelta
import re

# 匿名 S3 client
s3 = boto3.client('s3', config=Config(signature_version=UNSIGNED))
BUCKET = 'noaa-goes19'

# 方式 1: 查找最近一次全圆盘扫描的所有通道
def find_latest_scan(s3, bucket, product="ABI-L1b-RadF"):
    now = datetime.now(timezone.utc)
    for offset_hours in range(0, 6):
        check = now - timedelta(hours=offset_hours)
        year = check.strftime("%Y")
        doy = f"{check.timetuple().tm_yday:03d}"
        hour = check.strftime("%H")
        prefix = f"{product}/{year}/{doy}/{hour}/"
        resp = s3.list_objects_v2(Bucket=bucket, Prefix=prefix, MaxKeys=500)
        contents = resp.get('Contents', [])
        if not contents:
            continue
        # 按扫描时间分组
        scans = {}
        for c in contents:
            m = re.search(r's(\d{7})(\d{6})', c['Key'])
            if m:
                scans.setdefault(m.group(1)+m.group(2), []).append(c)
        if scans:
            latest = sorted(scans.keys())[-1]
            return scans[latest], latest
    return None, None

# 方式 2: 下载真彩色所需通道 (C01=蓝, C02=红, C03=近红外)
scan_files, scan_time = find_latest_scan(s3, BUCKET)
for f in scan_files:
    m = re.search(r'M6C(\d+)', f['Key'])
    if m and int(m.group(1)) in [1, 2, 3]:
        s3.download_file(BUCKET, f['Key'], f["./{f['Key'].split('/')[-1]}")
```

### 真彩色图生成（satpy）

GOES ABI 真彩色使用 C01（蓝, 0.47μm）、C02（红, 0.64μm）、C03（近红外, 0.86μm）三通道合成。

**关键问题：通道分辨率不同**
- C02（红）: 0.5km，shape=(21696, 21696)
- C01（蓝）: 1km，shape=(10848, 10848)
- C03（近红外）: 1km，shape=(10848, 10848)

**解决方案：手动降采样 C02（每2像素取1）**

```python
from satpy import Scene
import numpy as np
from PIL import Image

scn = Scene(reader='abi_l1b', filenames=[str(f) for f in data_dir.glob("*.nc")])
scn.load(['C01', 'C02', 'C03'])

# 手动降采样 C02 (0.5km -> 1km)
r_data = np.array(scn['C02'].values[::2, ::2])  # 21696 -> 10848
g_data = np.array(scn['C03'].values)
b_data = np.array(scn['C01'].values)

# 归一化 + Gamma 校正
def normalize(arr, pct_lo=2, pct_hi=98):
    lo, hi = np.nanpercentile(arr, [pct_lo, pct_hi])
    return np.clip((arr - lo) / (hi - lo), 0, 1)

rgb = np.dstack([normalize(r_data)**2.2, normalize(g_data)**2.2, normalize(b_data)**2.2])
rgb = np.flipud(rgb)  # satpy 数据上南下北

Image.fromarray((rgb * 255).astype(np.uint8)).save("goes19_truecolor.png")
```

## 适用场景

- 美洲区域静止卫星观测
- 飓风、强对流等高频次监测
- 真彩色图像合成
- 海表温度（SST）反演
- 气溶胶、云特性分析
- 火灾监测（ABI L2 FDCC）
- 闪电监测（GLM L2 LCFA）

## 不适用场景

- 东半球覆盖（应选 Himawari）
- 极地高纬度（静止卫星观测角度差）
- 实时性 < 5 分钟（受观测间隔限制）
- **可见光真彩色图在夜间使用**（无太阳光照，应用红外通道）

## 已知限制

### 1. pyspectral 依赖问题

satpy 的 `true_color` 复合通道依赖 `pyspectral` 库进行 Rayleigh 散射校正。若 pyspectral 安装失败，真彩色图边缘会出现紫色调。

**解决**：使用手动 RGB 合成（见上文代码），不依赖 satpy 内置 true_color。

### 2. 重采样内存不足

satpy 的 `resample()` 方法在处理 21696×21696 的 C02 通道时可能内存不足（>128MB 数组分配失败）。

**解决**：手动降采样（`[::2, ::2]`）替代 satpy resample，避免内存密集操作。

### 3. 图像方向

GOES 全圆盘原始数据不是北朝上，静止卫星投影特性导致图像有一定旋转。如需标准北朝上，需应用地图投影。
