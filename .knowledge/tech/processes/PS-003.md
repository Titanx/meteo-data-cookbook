## 背景

葵花 8/9（Himawari-8/9）是日本气象厅（JMA）运行的静止气象卫星，覆盖东亚-澳洲区域（80°E-160°W, 60°N-60°S），提供 10 分钟间隔的高频观测，16 个波段（可见光、近红外、红外），是东亚区域最常用的静止卫星数据源。

数据获取有两种途径：
- **AWS S3 匿名访问**（推荐，无需注册） - NOAA 在 AWS Open Datasets 提供
- **JAXA P-Tree FTP**（需要注册） - JAXA 官方完整数据

本条目基于 2026-07-18 的实际测试验证，**AWS S3 匿名访问完全成功，已实际下载文件**。

### 文件命名规则

**L1b 文件**：`HS_H08_{YYYYMMDD}_{HHMM}_B{XX}_FLDK_R{RR}_S{PPPP}.DAT.bz2`

| 字段 | 含义 | 示例 |
|------|------|------|
| `HS` | Himawari Standard | `HS` |
| `H08` | 卫星编号 | `H08`（AWS 上仍用 H08） |
| `YYYYMMDD` | 日期（UTC） | `20251126` |
| `HHMM` | 时分（UTC） | `0000` |
| `B{XX}` | 波段编号（B01-B16） | `B01`（可见光蓝） |
| `FLDK` | 区域 | `FLDK`（全圆盘） |
| `R{RR}` | 位深 | `R10`（10bit）/ `R20`（20bit） |
| `S{PPPP}` | 像素分块编号 | `S0110`（第1块） |

**L2 文件**：`NC_H08_{YYYYMMDD}_{HHMM}_L2_{PRODUCT}.nc`（NetCDF 格式）

### Python 下载示例

```python
import boto3
from botocore import UNSIGNED
from botocore.config import Config

# 创建匿名 S3 client
s3 = boto3.client('s3', config=Config(signature_version=UNSIGNED))

bucket = "noaa-himawari8"

# 方式 1: 下载指定文件
key = "AHI-L1b-FLDK/2025/11/26/0000/HS_H08_20251126_0000_B01_FLDK_R10_S0110.DAT.bz2"
s3.download_file(bucket, key, "himawari_b01.dat.bz2")

# 方式 2: 列出指定时间的所有波段
import re
prefix = "AHI-L1b-FLDK/2025/11/26/0000/"
response = s3.list_objects_v2(Bucket=bucket, Prefix=prefix)
for obj in response.get('Contents', []):
    print(f"{obj['Key']}  ({obj['Size']/1024/1024:.2f} MB)")

# 方式 3: 下载完整全圆盘所有波段（16 个波段 × 10 个分块 = 160 个文件）
target_time = "2025/11/26/0000/"
prefix = f"AHI-L1b-FLDK/{target_time}"
response = s3.list_objects_v2(Bucket=bucket, Prefix=prefix)
for obj in response.get('Contents', []):
    s3.download_file(bucket, obj['Key'], f"./{obj['Key'].split('/')[-1]}")
```

### FLDK 分段下载策略（区域应用优化）

全圆盘 FLDK 数据每波段分 10 个纬度分段（S0110-S1010），每个分段覆盖约 12° 纬度。**对于区域应用，只需下载覆盖目标区域的分段**，可节省 70-80% 下载量。

#### 分段纬度映射

| 分段号 | 覆盖纬度 | 适用区域 |
|--------|---------|---------|
| S0110 | 60°N-48°N | 西伯利亚 |
| S0210 | 48°N-36°N | 华北、东北 |
| S0310 | 36°N-24°N | 华东、华中、华南 |
| S0410 | 24°N-12°N | 东南亚 |
| S0510-S1010 | 12°N-60°S | 赤道、南半球 |

#### 华东区域下载方案（已验证）

华东区域（114-123°E, 23-38°N）只需 **S0210 + S0310** 两个分段：

| 方案 | 文件数/时次 | 下载量/时次 | 说明 |
|------|-----------|-----------|------|
| 全圆盘 | 30 | ~454 MB | B01/B02×10段 + B03×10段 |
| **分段（推荐）** | **6** | **~100 MB** | B01/B02/B03 × S0210+S0310 |
| 节省 | 80% | 78% | |

#### satpy 分段处理机制

satpy 的 `GEOSegmentYAMLReader`（ahi_hsd reader 使用）支持部分分段加载：
- `pad_data=True`（默认）：自动用 NaN 填充缺失分段，构建全圆盘几何
- 文件名中的 `S{segment:2d}{total_segments:2d}` 告诉 satpy 总分段数
- 即使只提供 2/10 个分段，area_extent 仍正确计算为全圆盘范围
- 重采样时只提取目标区域，NaN 分段不影响结果

```python
from satpy import Scene
from pyresample.geometry import AreaDefinition

# 只下载 S0210 + S0310 的 6 个文件
files = sorted(glob('HS_H09_*_B0[123]_FLDK_*_S02[01]0.DAT.bz2'))
scn = Scene(filenames=files, reader='ahi_hsd')
scn.load(['B01', 'B02', 'B03'])

# satpy 自动填充其余8个分段为NaN，area_extent为全圆盘
# 重采样到华东区域 - 只提取有数据的部分
east_china = AreaDefinition(
    'east_china', 'East China', 'east_china',
    {'proj': 'longlat', 'datum': 'WGS84', 'ellps': 'WGS84'},
    900, 1500, (114.0, 23.0, 123.0, 38.0))
new_scn = scn.resample(east_china, resampler='nearest', radius_of_influence=5000)
```

#### 白天时次选择（关键！）

可见光通道（B01/B02/B03）需要太阳光照，**必须选择白天时次**：

| 目标区域 | 推荐UTC时间 | 北京时间 | 太阳高度角 |
|---------|-----------|---------|-----------|
| 华东（114-123°E） | 02:00-05:00 | 10:00-13:00 | 最佳 |
| 华南（105-118°E） | 03:00-06:00 | 11:00-14:00 | 良好 |
| 全国（80-130°E） | 04:00-06:00 | 12:00-14:00 | 折中 |

**夜间时次（如UTC 17:00 = 北京01:00）可见光通道无信号，图像全为噪声。**

#### Japan Area 不适用于华东

Japan Area（JP01-JP04）文件虽小（B01约1MB），但实际覆盖范围仅日本附近（约137-142°E），**不覆盖华东区域**（114-123°E）。satpy 加载 Japan Area 数据后华东区域为 NaN。

## 方式二：JAXA P-Tree FTP（完整数据，需注册）

### 适用场景

- 需要 Himawari-9 主星数据（AWS 可能延迟更新 H09 文件名）
- 需要日本区域（Japan）和目标区域（Target）的高频数据
- 需要 JAXA 官方处理的高级产品
- 业务系统需要稳定 SLA

### 注册流程

1. 访问 https://www.eorc.jaxa.jp/ptree/registration_top.html
2. 填写姓名、机构、邮箱、用途
3. 等待邮件（1-3 工作日）
4. 获取 FTP 凭证：`ftp.ptree.jaxa.jp`

### 与 AWS S3 的对比

| 维度 | AWS S3（推荐） | JAXA FTP |
|------|--------------|----------|
| 注册要求 | 无需注册 | 需注册审核 |
| 认证 | 匿名 | 账号密码 |
| 协议 | HTTPS / S3 API | FTP |
| 数据覆盖 | 2015-2025 | 2015 至今 |
| 数据完整性 | L1b + 部分 L2 | 全部产品 |
| 工具支持 | boto3 / aws-cli / HTTP | ftplib / FTP 客户端 |
| 网络要求 | HTTPS 443 | FTP 21 |
| 速度 | AWS CDN 加速 | 受日本网络影响 |

## 数据解码处理

下载的 `.DAT.bz2` 文件需要：
1. **解压**：`bzip2 -d file.DAT.bz2` 或 Python `bz2` 库
2. **读取 HSD**：使用 `satpy` 库

```python
from satpy import Scene

# 加载 Himawari HSD 文件
files = ["HS_H08_20251126_0000_B01_FLDK_R10_S0110.DAT"]
scn = Scene(reader='ahi_hsd', filenames=files)
scn.load(['true_color'])

# 保存为 PNG
scn.save_dataset('true_color', 'himawari_truecolor.png')
```

## 适用场景

- 东亚-澳洲区域静止卫星观测
- 台风、强对流等高频次监测
- 真彩色图像合成
- 海表温度（SST）反演
- 气溶胶、云特性分析
- 科研与教育（AWS S3 零门槛）

## 不适用场景

- 全球覆盖（应选 GOES、Meteosat）
- 极地高纬度（静止卫星观测角度差）
- 实时性 < 10 分钟（受观测间隔限制）
- **可见光真彩色图在夜间使用**（UTC 17:00-22:00 对应北京 01:00-06:00，无太阳光照，图像全噪声。应用红外通道或选择白天时次）
