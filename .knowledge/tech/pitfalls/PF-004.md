# [PF-004] Meteostat 区域数据下载陷阱（中国/亚洲/美洲）

> **成熟度**: verified

> **最后引用**: 2026-07-20

> **适用阶段**: implement, verify

## 背景

Meteostat 聚合全球气象站数据，但不同区域站点密度和数据质量差异显著。2026-07-19~20 在下载全球 246 个机场气象数据时（中国46 + 东亚东南亚115 + 南北美洲85），遇到多个与区域特殊性相关的问题。本条目记录这些陷阱及验证过的解决方案。

## 陷阱列表

### 陷阱 1: 默认搜索半径太小导致"无站点"

**现象**：用 `Stations().nearby(lat, lon, radius=30000)` 查询中国西部/部分东部机场时返回空结果，误判为"该地区无数据"。

**实际原因**：Meteostat 在中国西部站点密度极低，30km 内常无站点。但扩大到 100-200km 后通常能找到。

**解决方案**：逐级扩大半径搜索

```python
def find_station_robust(lat, lon):
    for radius_km in [30, 50, 100, 200, 500]:
        df = Stations().nearby(lat, lon, radius=radius_km * 1000).fetch()
        if len(df) > 0:
            return df.index[0], df.iloc[0]
    return None, None
```

**实测案例**：
- ZSNB 宁波：30/50km 无结果 → 100km 找到 58477 Dinghai
- ZLLL 兰州：30/50/100km 无结果 → 200km 找到 52889 Lanzhou

### 陷阱 2: 多机场映射到同一站点

**现象**：同一城市的多个机场（如北京首都/大兴、上海浦东/虹桥）返回完全相同的数据，看似数据重复。

**实际原因**：Meteostat 一个城市通常只有一个站点，多个机场都映射到该站点。

**影响**：数据分析时需去重，否则会重复计数。

**实测案例**：
| 站点ID | 站点名 | 共享机场 |
|--------|--------|---------|
| 54511 | Beijing | ZBAA首都, ZBAD大兴 |
| 58367 | Shanghai/Hongqiao | ZSPD浦东, ZSSS虹桥 |
| 45011 | Taipa | ZGSD珠海, VMMC澳门 |

**解决方案**：保存 `stations_metadata.csv` 记录映射关系，分析时按 station_id 去重。

### 陷阱 3: 部分站点年度数据不完整

**现象**：2025年全年下载时，部分站点只返回161天数据（约6月后），而非365天。

**实际原因**：这些站点的数据源（如港澳台地区的自动站）在 Meteostat 中更新延迟或仅提供半年数据。

**实测案例**：
| 站点ID | 站点 | 2025年数据天数 | 原因 |
|--------|------|--------------|------|
| 46689 | Mazu（福州代用） | 161天 | 马祖站数据源延迟 |
| 57679 | Changsha | 161天 | 长沙站更新延迟 |
| 45001 | Cheung Chau（香港代用） | 161天 | 香港站数据源问题 |
| 46697 | Taoyuan Ab（台北代用） | 161天 | 台北站数据源问题 |

**解决方案**：
- 下载后检查记录数，与预期天数（365/366）对比
- 对不完整站点，考虑用 Open-Meteo ERA5 再分析数据补充
- 记录到 `download_summary.csv` 的 `daily_count` 字段便于筛查

### 陷阱 4: 站点距离远导致代表性下降

**现象**：某些机场映射的站点距离很远（如拉萨46km、福州36km），数据可能不能准确代表机场位置气象。

**实测案例**：
| 机场 | 映射站点 | 距离 | 影响 |
|------|---------|------|------|
| ZULS 拉萨 | 55591 Lhasa | 46km | 拉萨贡嘎机场在山谷，市区站在高原，地形差异大 |
| ZSFZ 福州 | 46689 Mazu | 36km | 马祖是海岛站，与福州内陆气候有差异 |
| ZBAD 北京大兴 | 54511 Beijing | 48km | 北京站位于市区，大兴机场在郊区 |

**解决方案**：
- 在 `stations_metadata.csv` 中记录 `distance_km` 字段
- 分析时标注站点距离，距离>30km 的结果需谨慎解读
- 对精度要求高的场景，使用 Open-Meteo 按经纬度插值的再分析数据

### 陷阱 5: 文件命名包含站点ID导致管理混乱

**现象**：早期脚本用 `daily_{ICAO}_{station_id}.csv` 命名，当站点ID变更（如重试后换站）时产生冗余文件。

**解决方案**：统一用 `daily_{ICAO}.csv` 命名，station_id 记录在元数据文件中。

```python
# ❌ 避免
daily_ZSNB_58477.csv  # 站点ID变更后会留下旧文件

# ✅ 推荐
daily_ZSNB.csv  # 始终覆盖，站点ID记录在stations_metadata.csv
```

## 亚洲扩展下载陷阱（2026-07-19）

### 陷阱 6: 缅甸日数据严重缺失（小时数据正常）

**现象**：缅甸 6 个机场（仰光、曼德勒、内比都、密支那、土瓦、丹老等）2026 年日数据仅 31-38 天，但小时数据完整（4755-4800 条）。

**实际原因**：疑 Meteostat 日数据聚合接口对缅甸站点存在 bug，无法将小时数据正确聚合为日数据。

**解决方案**：从小时数据手动聚合日数据（`resample('D')`），或使用 Open-Meteo ERA5 补充。

### 陷阱 7: 越南部分机场完全无数据

**现象**：VVNB 河内、VVBM 邦美蜀、VVDL 大叻三个机场虽然有映射站点，但返回 0 条数据。

**实际原因**：
- VVNB 河内映射到 VVGL0 (Hanoi/Xóm Pho)，该站点未收录观测数据
- VVBM 和 VVDL 共用同一站点 VVDL0，该站点无数据

**解决方案**：扩大半径搜索替代站点，或使用 Open-Meteo 按经纬度获取 ERA5 再分析数据。

### 陷阱 8: 蒙古站点距离远且部分无数据

**现象**：
- ZMMN 木伦映射到 30925 Kjahta（俄罗斯边境），距离 70km
- ZMHB 科布多映射到 44224 Khovd，站点存在但无观测记录
- ZMUB 乌兰巴托日数据为 0（小时数据正常）

**解决方案**：蒙古地区数据质量较差，建议结合 Open-Meteo 补充。

### 陷阱 9: 日本德岛/高松共用同一站点

**现象**：RJOS 德岛和 RJOT 高松都映射到站点 47880 (Takamatsu Airport)，返回完全相同的数据。

**影响**：两机场距离约65km，数据不能分别代表各自位置。

### 陷阱 10: 东帝汶完全无数据

**现象**：WPDL 帝力虽然映射到 97390 Dilli，但该站点无任何观测记录。

**解决方案**：东帝汶地区使用 Open-Meteo ERA5 再分析数据。

## 美洲下载陷阱（2026-07-20）

### 陷阱 11: 美洲整体数据质量优于亚洲

**对比结论**：南北美洲 85 个机场 100% 成功，无任何失败案例。美国 20 个主要机场全部在 5km 内找到站点，数据完整度极高（365天完整）。

**启示**：Meteostat 数据源主要来自 NOAA ISD，美洲（尤其是美国）站点密度和数据质量最高。亚洲和非洲地区数据质量明显下降。

### 陷阱 12: 中美洲部分机场年度数据不完整

**现象**：
- MGGT 危地马拉城：2025年仅161天数据（约6月后停止更新）
- SGES 东方城（巴拉圭）：2025年仅169天数据

**原因**：中美洲部分站点数据源更新延迟。

### 陷阱 13: 累西腓降水数据异常为0

**现象**：SBRF 累西腓（巴西）2025 和 2026 年降水均为 0mm，但该城市年降水量应超过 2000mm。

**原因**：站点降水传感器可能故障或数据未上报。

**解决方案**：降水数据需交叉验证，单站异常值应剔除或用邻近站插值。

## 验证方法

### 数据完整性检验脚本

使用 `check_data_integrity.py` 从4个维度检验：
1. **时间覆盖**：实际记录数 vs 预期天数（365/200）
2. **字段缺失率**：核心字段（tavg/tmin/tmax/temp/rhum/pres）缺失率
3. **异常值检测**：温度-15~45°C、湿度0-100%、气压950-1060hPa 范围检查
4. **站点一致性**：检测多机场共享站点情况

### 质量评分标准

| 维度 | 优 | 良 | 中 | 差 |
|------|---|---|---|---|
| 时间覆盖 | ≥99% | ≥95% | ≥90% | <90% |
| 核心字段缺失 | <1% | <5% | <20% | ≥20% |
| 异常值 | 0个 | ≤2个 | ≤5个 | >5个 |
