# [PF-005] ERCOT 官网反爬虫屏蔽与中国 IP 不可访问陷阱

> **类型**: pitfall
> **成熟度**: verified
> **创建时间**: 2026-07-23
> **最后引用**: 2026-07-23
> **标签**: ercot, texas, electricity, price, imperva, incapsula, anti-scraping, china-ip-block, eia-api, gridstatus
> **适用阶段**: architect, implement

## 背景

ERCOT（Electric Reliability Council of Texas）是德州独立系统运营商，管理德州 90% 负荷，其电力市场数据（DAM/RTM 结算点电价、负荷、发电燃料类型等）对电力市场研究和极端天气-电力联动分析极有价值。2026-07-22~23 在尝试下载 ERCOT 历史电价数据时，遇到了一系列数据获取障碍，本条目记录这些陷阱及验证过的替代方案。

## 陷阱列表

### 陷阱 1: ERCOT 官网对中国 IP 完全屏蔽

**现象**：从中国 IP 访问 `www.ercot.com`、`data.ercot.com`、`mis.ercot.com` 全部无法打开，浏览器直接超时或返回空白页。

**实际原因**：ERCOT 部署了 Imperva Incapsula 反爬虫系统，对中国 IP 段实施了地域级屏蔽。这不是普通的 JS challenge，而是 IP 级别的完全封锁。

**验证过程**：
- 纯 `requests` 库调用 MIS API → 403
- 带浏览器 User-Agent 的 `requests` → 403
- `cloudscraper` 库（专门绕过反爬虫）→ 403
- 浏览器自动化（MCP browser tools）先访问主页再 fetch API → 403
- 用户手动用浏览器打开网页 → 无法打开

**结论**：从中国 IP 无法以任何方式获取 ERCOT 官网数据。

### 陷阱 2: gridstatus 开源库因 Python 版本不可用

**现象**：`gridstatus` 库（v0.36.0，2026-04 发布）支持 ERCOT，但安装后 `import gridstatus` 报 `StrEnum` 错误。

**实际原因**：gridstatus v0.32+ 使用了 Python 3.11 的 `StrEnum`，而项目环境是 Python 3.10.11。

**解决方案**：降级到 `gridstatus==0.31.0`（2025-08 发布），兼容 Python 3.10。但降级后 ERCOT 接口仍返回 403（因为底层仍访问 ercot.com）。

### 陷阱 3: EIA API 不提供电价数据

**现象**：EIA（美国能源信息署）API v2 提供丰富的 ERCOT 运行数据，容易误以为也包含电价。

**实际原因**：EIA API v2 的 `electricity/rto/` 路由只有以下数据：
- `region-data`：负荷（D）、日前负荷预测（DF）、净发电量（NG）、总交换量（TI）
- `fuel-type-data`：按燃料类型的发电量（Solar/Wind/Coal/NG/Nuclear/Battery/Hydro/Other/UES）
- `region-sub-ba-data`：子区域负荷
- `interchange-data`：邻区交换

**不含 LMP/SPP 等任何电价数据**。EIA ICE 批发电力价格 Excel（`ice_electric-*.xlsx`）也不包含 ERCOT 枢纽。

### 陷阱 4: HB_PANHANDLE 枢纽无数据

**现象**：从 GridStatus.io 下载 ERCOT 5 个交易枢纽电价时，HB_PANHANDLE 返回空数据。

**实际原因**：HB_PANHANDLE 枢纽可能已被弃用或重组到其他枢纽。ERCOT 的交易枢纽随市场演化有调整。

**解决方案**：下载时对返回空数据的枢纽跳过即可。实际可用枢纽为 HB_NORTH、HB_SOUTH、HB_HOUSTON、HB_WEST 共 4 个。

## 替代方案

| 方案 | 电价 | 负荷/发电 | 中国可访问 | 免费 | 推荐 |
|------|------|----------|-----------|------|------|
| ERCOT 官网 MIS | DAM/RTM SPP | 全部 | 否 | 是 | 否 |
| EIA API v2 | 无 | 有 | 是 | 是 | 是（负荷/发电） |
| GridStatus.io API | DAM/RTM SPP | 有 | 是 | 免费方案 | 是（电价） |
| GridStatus 开源库 | 理论有 | 有 | 否（访问 ERCOT） | 是 | 否 |

## 经验教训

1. **遇到 403 先确认是否地域屏蔽**：Imperva 返回 403 可能是 JS challenge（可绕过），也可能是 IP 级屏蔽（不可绕过）。从不同网络环境测试可区分。
2. **EIA API 不是万能的**：EIA 聚合了美国各 ISO 的运行数据，但不包含任何 ISO 的电价数据。电价需要从 ISO 或第三方获取。
3. **GridStatus.io 免费方案足够研究用途**：每月 250 次请求 / 50 万行，下载 ERCOT 4 个枢纽 1.5 年 DAM+RTM 仅用 16 次请求 / 27 万行。
4. **API key 安全**：GridStatus.io 和 EIA 的 API key 是个人凭证，不应硬编码在脚本中或提交到版本库，应通过环境变量传入。
