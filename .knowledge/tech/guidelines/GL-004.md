## 背景

气象数据获取常受制于：API key 申请繁琐、商业授权昂贵、数据源不稳定。Open-Meteo 作为开源天气 API，提供免费、无 key、非商业用途开放的访问方式，集成了 ERA5、IFS、GFS 等多家国家气象中心的模式输出，是科研验证、原型开发、小规模业务的理想选择。本条目基于 2026-07-18 的实际测试验证。

## 内容

### 推荐做法（recommend）

#### 1. API 概览

| 端点 | 用途 | 数据源 | 时间范围 |
|------|------|--------|---------|
| `https://api.open-meteo.com/v1/forecast` | 实时预报 | GFS/ICON/ECMWF 等 | 未来 7-16 天 |
| `https://archive-api.open-meteo.com/v1/archive` | 历史再分析 | ERA5/ERA5-Land/IFS | 1940 至今 |

**核心特点**：
- 无需 API key（非商业用途，每日 < 10,000 次调用）
- 支持 JSON / CSV / XLSX 三种输出格式
- 自动选择最优模式（Best Match），也可手动指定
- 支持时区转换，避免 UTC 转换错误
- 支持多坐标批量请求（逗号分隔）

#### 2. 实时预报 API 调用模板

```python
import requests
import pandas as pd

url = "https://api.open-meteo.com/v1/forecast"
params = {
    "latitude": 39.9042,   # 北京
    "longitude": 116.4074,
    "hourly": [
        "temperature_2m",
        "relative_humidity_2m",
        "wind_speed_10m",
        "wind_direction_10m",
        "precipitation",
        "surface_pressure",
    ],
    "daily": [
        "temperature_2m_max",
        "temperature_2m_min",
        "precipitation_sum",
    ],
    "timezone": "Asia/Shanghai",  # 自动时区转换
    "forecast_days": 7,
}

response = requests.get(url, params=params, timeout=30)
data = response.json()

# 转为 DataFrame
hourly_df = pd.DataFrame({
    "time": data["hourly"]["time"],
    "temp_2m_C": data["hourly"]["temperature_2m"],
    "rh_2m_pct": data["hourly"]["relative_humidity_2m"],
})
hourly_df["time"] = pd.to_datetime(hourly_df["time"])
```

#### 3. 历史 ERA5 数据调用模板

```python
url = "https://archive-api.open-meteo.com/v1/archive"
params = {
    "latitude": 39.9042,
    "longitude": 116.4074,
    "start_date": "2026-07-01",   # ISO8601 格式
    "end_date": "2026-07-17",
    "hourly": [
        "temperature_2m",
        "relative_humidity_2m",
        "dew_point_2m",
        "precipitation",
        "wind_speed_10m",
        "wind_direction_10m",
        "surface_pressure",
    ],
    "daily": [
        "temperature_2m_max",
        "temperature_2m_min",
        "temperature_2m_mean",
        "precipitation_sum",
    ],
    "timezone": "Asia/Shanghai",
    "model": "era5",   # 可选 era5 / era5_land / era5_seamless / ifs
}

response = requests.get(url, params=params, timeout=60)
data = response.json()
```

#### 4. 可用模型与分辨率

| 模型 | 分辨率 | 时间范围 | 适用场景 |
|------|--------|---------|---------|
| ERA5 | 0.25° (~25km) | 1940 至今 | 气候研究，长期一致 |
| ERA5-Land | 0.1° (~11km) | 1950 至今 | 地面要素，高分辨率 |
| ERA5-Seamless | 混合 | 1940 至今 | 温湿度用 ERA5-Land，风和辐射用 ERA5 |
| ECMWF IFS | 9km | 2017 至今 | 近期高精度 |
| CERRA | 5km | 1985-2021 | 欧洲区域 |

#### 5. 关键参数说明

| 参数 | 说明 |
|------|------|
| `latitude`, `longitude` | WGS84 坐标，支持逗号分隔多坐标 |
| `elevation` | 手动设置海拔，用于山地降尺度 |
| `start_date`, `end_date` | ISO8601 格式 `YYYY-MM-DD` |
| `hourly` / `daily` | 要素列表，逗号分隔 |
| `timezone` | 时区名（如 `Asia/Shanghai`），默认 GMT |
| `temperature_unit` | `celsius` 或 `fahrenheit` |
| `wind_speed_unit` | `kmh` / `ms` / `mph` / `kn` |
| `precipitation_unit` | `mm` 或 `inch` |
| `timeformat` | `iso8601` / `unixtime` |
| `cell_selection` | `land` / `sea` / `nearest` |
| `model` | 历史数据专用，指定再分析模型 |

### 禁止做法（avoid）

- ❌ 商业用途未购买商业授权（需联系 Open-Meteo）
- ❌ 每日调用超过 10,000 次（免费额度限制）
- ❌ 未设置 `timeout` 导致请求挂起
- ❌ 未指定 `timezone` 导致时间偏移
- ❌ 长时间序列（>1 年）单次请求，应分批拉取
- ❌ 未保存原始 JSON，仅存转换后的 CSV（丢失元数据）

## 适用场景

- 科研项目的气象数据获取（无需申请 API key）
- 原型开发与快速验证
- 气象数据可视化的小规模数据源
- 历史天气回溯分析（ERA5 1940 至今）
- 替代付费 API 的低成本方案

## 不适用场景

- 商业产品（需购买商业授权）
- 大规模业务系统（每日 > 10,000 次调用）
- 实时性要求 < 1 分钟的场景（API 有更新延迟）
- 需要原始模式输出（应直接访问 GFS/ECMWF）
