## 背景

NASA POWER（Prediction Of Worldwide Energy Resources）是 NASA Langley Research Center 维护的全球气象与太阳能数据服务，前身为 1997 年的 SSE（Surface meteorology and Solar Energy）项目，现为 POWER Data v10（云原生架构）。它提供**多源同化**的气象和辐射数据，不同于纯再分析（ERA5）或纯实测（SURFRAD），是卫星反演 + 再分析的混合产品。

本条目基于 2026-07-20 的实际 API 测试验证，并与 SURFRAD 实测数据做了精度对比。

## 内容

### 推荐做法（recommend）

#### 1. 数据源概览

| 维度 | 说明 |
|------|------|
| 数据来源 | 多源同化：CERES 卫星反演（辐射）+ MERRA-2 再分析（气象）+ GPM IMERG（降水） |
| 访问方式 | REST API（HTTPS） |
| API key | **不需要** |
| 注册 | **不需要** |
| 时间范围 | 气象 1981 起，辐射 1984 起，小时数据 2001 起 |
| 空间覆盖 | 全球 |
| 空间分辨率 | 辐射 1°×1°，气象 0.5°×0.625°，降水 0.1°×0.1° |
| 数据频率 | 小时、日、月、气候态 |
| 授权 | 完全免费，无商业限制 |
| 当前版本 | POWER Data v10 |

#### 2. 数据来源分层（关键！）

POWER 不是单一数据源，不同参数来自不同源，延迟和精度各异：

| 参数类型 | 数据源 | 方法 | 延迟 |
|---------|--------|------|------|
| 辐射（GHI/DNI/DHI） | CERES SYN1deg Ed4.2 + FLASHFlux v4A | 卫星观测反演 | **3-4 个月**（SYN1deg），5-7 天（FLASHFlux NRT） |
| 气象（温度/风速/湿度/气压） | MERRA-2 / GEOS-IT | 再分析同化 | **2-3 天**（GEOS-IT） |
| 降水 | GPM IMERG Final / Late | 卫星反演 | 14 小时（Late），3.5 个月（Final） |

**重要**：辐射数据延迟远大于气象数据。实测发现 2026-07-01 的辐射数据仍返回 -999（缺失），需用 2025 年数据才能获取完整辐射。

#### 3. 三大社区（community 参数）

| 社区代码 | 名称 | 适用场景 | 特有参数 |
|---------|------|---------|---------|
| `RE` | Renewable Energy | 光伏/光热 | DNI、DHI、晴空辐照 |
| `SB` | Sustainable Buildings | 建筑能耗 | EPW 格式输出、度日数 |
| `AG` | Agroclimatology | 农业 | ICASA 格式、PAR、ET0 |

#### 4. API 端点结构

```
https://power.larc.nasa.gov/api/temporal/{temporal}/{spatial}?parameters=...&community=...&longitude=...&latitude=...&start=...&end=...&format=...
```

| 参数 | 必需 | 说明 |
|------|------|------|
| `temporal` | 是 | `hourly` / `daily` / `monthly` / `climatology` |
| `spatial` | 是 | `point` / `regional`（hourly 仅支持 point） |
| `parameters` | 是 | 参数名列表，逗号分隔，**最多 15 个** |
| `community` | 是 | `RE` / `SB` / `AG` |
| `longitude`, `latitude` | 是 | 经纬度 |
| `start`, `end` | 是 | YYYYMMDD |
| `format` | 是 | `CSV` / `JSON` / `ASCII` / `NetCDF` / `ICASA` / `EPW` |
| `time-standard` | 否 | `UTC` 或 `LST`（仅 hourly，默认 LST） |

#### 5. 小时数据获取模板

```python
import requests
import pandas as pd
from io import StringIO

# Bondville, IL
LAT, LON = 40.05, -88.37

url = (f"https://power.larc.nasa.gov/api/temporal/hourly/point?"
       f"parameters=T2M,ALLSKY_SFC_SW_DWN,ALLSKY_SFC_SW_DNI,ALLSKY_SFC_SW_DIFF,"
       f"WS10M,RH2M,PS&community=RE&longitude={LON}&latitude={LAT}"
       f"&start=20250701&end=20250710&format=CSV&header=true&time-standard=UTC")

resp = requests.get(url, timeout=60)

# POWER CSV 有多行 header，需找到 -END HEADER- 后的数据
lines = resp.text.split('\n')
data_start = 0
for i, line in enumerate(lines):
    if '-END HEADER-' in line:
        data_start = i + 1
        break

df = pd.read_csv(StringIO('\n'.join(lines[data_start:])))
# 列: YEAR, MO, DY, HR, T2M, ALLSKY_SFC_SW_DWN, ...
```

#### 7. 核心参数字段

**温度类（MERRA-2）**：

| 参数 | 含义 | 单位 |
|------|------|------|
| `T2M` | 2 米气温 | °C |
| `T2M_MAX` / `T2M_MIN` | 日最高/最低气温 | °C |
| `T2MDEW` | 2 米露点温度 | °C |
| `T2MWET` | 2 米湿球温度 | °C |
| `TS` | 地表温度 | °C |

**辐照类（CERES/SRB）**：

| 参数 | 含义 | 单位（hourly） | 单位（daily） |
|------|------|---------------|--------------|
| `ALLSKY_SFC_SW_DWN` | **GHI** 全天空短波下行 | Wh/m²（= W/m²） | kW·h/m²/day |
| `ALLSKY_SFC_SW_DNI` | **DNI** 直接法向 | Wh/m² | kW·h/m²/day |
| `ALLSKY_SFC_SW_DIFF` | **DHI** 散射 | Wh/m² | kW·h/m²/day |
| `CLRSKY_SFC_SW_DWN` | 晴空 GHI | Wh/m² | kW·h/m²/day |
| `ALLSKY_SFC_UV_INDEX` | 紫外指数 | 无量纲 | — |
| `ALLSKY_SFC_PAR_TOT` | 光合有效辐射 | W/m² | — |
| `SZA` | 太阳天顶角 | 度 | — |

**⚠️ 单位陷阱**：hourly 的 GHI 单位是 **Wh/m²**（1 小时累计，数值上等于 W/m²），daily 的单位是 **kW·h/m²/day**。API header 会标注实际单位，务必读取确认。

**风速**：

| 参数 | 含义 | 单位 |
|------|------|------|
| `WS2M` / `WS10M` / `WS50M` | 2/10/50 米风速 | m/s |
| `WD10M` | 10 米风向 | 度 |

**湿度气压**：

| 参数 | 含义 | 单位 |
|------|------|------|
| `RH2M` | 2 米相对湿度 | % |
| `QV2M` | 2 米比湿 | g/kg |
| `PS` | 地表气压 | kPa |

**降水**：

| 参数 | 含义 | 单位（hourly） | 单位（daily） | 说明 |
|------|------|---------------|--------------|------|
| `PRECTOT` | 总降水 | mm/hour | mm/day | 原始降水 |
| `PRECTOTCORR` | 校正降水 | — | mm/day | **仅 daily，UTC** |

**⚠️ 降水单位陷阱**：`PRECTOT` 在 hourly 接口单位是 **mm/hour**（小时降水速率），在 daily 接口单位是 **mm/day**（日累计）；`PRECTOTCORR` 仅 daily 可用。

#### 8. 参数完整清单（官方 API 查询，2026-07-21）

**关键发现**：POWER 有官方参数查询端点，无需猜测参数名。

```
https://power.larc.nasa.gov/api/system/manager/parameters
  ?user=nasapower4r&community=RE&temporal=HOURLY&metadata=true
```

**参数总数**（AG/RE/SB 三社区完全一致）：

| 时间分辨率 | 参数数 | 说明 |
|-----------|--------|------|
| HOURLY | 105 | 小时尺度核心参数 |
| DAILY | 152 | 含 max/min/range 统计 |
| MONTHLY | 1388 | 含大量太阳能工程衍生参数 |
| CLIMATOLOGY | 1634 | 2001-2020 气候态统计 |
| **不重复总计** | **1660** | — |

**HOURLY 105 个核心参数按类别分布**（合计 6+5+3+13+9+13+10+12+5+3+1+2+13+4+5+1=105 ✓）：

| 类别 | 数量 | 代表参数 |
|------|------|---------|
| 温度 | 6 | `T2M` `T10M` `T2MDEW` `T2MWET` `TS` `TS_ADJ` |
| 湿度水汽 | 5 | `RH2M` `QV2M` `QV10M` `DISPH` `RHOA` |
| 气压 | 3 | `PS` `PSC` `SLP` |
| 风 | 13 | `WS/WD/U/V` × {2m,10m,50m} + `WSC` |
| 降水蒸发 | 9 | `PRECTOT` `PRECTOTCORR` `PRECSNO` `EVLAND` `EVPTRNS` `GWETPROF/ROOT/TOP` |
| 辐射-全天空 | 13 | `ALLSKY_SFC_SW_DWN/DNI/DIFF/DIRH/UP` `LW_DWN/UP` `PAR_TOT/DIFF/DIRH` `UVA/UVB/UV_INDEX` |
| 辐射-晴空 | 10 | `CLRSKY_SFC_SW_DWN/DNI/DIFF/DIRH/UP` `LW_DWN/UP` `PAR_TOT/DIFF/DIRH` |
| 辐射-其他 | 12 | `TOA_SW_DWN/DNI` `LWLAND/SWLAND` `KT/NKT`(全天空/晴空) `SRF_ALB`(全天空/晴空/校正) `AIRMASS` |
| 云 | 5 | `CLOUD_AMT` `CLOUD_BT` `CLOUD_TT` `CLOUD_OD` `CLOUD_PHASE` |
| 气溶胶 | 3 | `AOD_55` `AOD_55_ADJ` `AOD_84` |
| 太阳几何 | 1 | `SZA` |
| 冰雪 | 2 | `FRSEAICE` `FRSNO` |
| 土壤 | 13 | `TSOIL1-6`(6) `SFMC/RZMC/PRMC`(3) + 百分位(3) `SNODP`(1) |
| 对流层/边界层 | 4 | `TROPPB` `TROPQ` `TROPT` `PBLTOP` |
| 大气柱总量 | 5 | `PW`(可降水) `TQV`(柱可降水) `TO3`(臭氧柱) `TSURF`(地表温度) `Z0M`(粗糙度) |
| 其他 | 1 | `ORIGINAL_ALLSKY_SFC_SW_*`(未校正辐射，2 个同类合并) |

**DAILY 额外 48 个参数**（在 HOURLY 基础上新增）：
- 统计量：`T2M_MAX/MIN/RANGE`、`WS10M_MAX/MIN/RANGE` 等 16 个
- 度日：`HDD0/10/18_3`（采暖）、`CDD0/10/18_3`（制冷）
- 生长度日：`GDD4.4/7.2/10/13.3/15.6`（农业）
- 天数：`FROST_DAYS` `CLRSKY_DAYS`
- IMERG 降水：`IMERG_PRECTOT` `IMERG_PRECLIQUID_PROB`
- 昼夜云量：`CLOUD_AMT_DAY/NIGHT`

**MONTHLY/CLIMATOLOGY 参数爆炸原因**（1388/1634 个）：
- `SI_TILTED` × {AVG/MAX/MIN} × {水平/纬度/纬度±15°/最优/垂直} = 30+ 个
- `SI_TRACKER` × {AVG/MAX/MIN} × {双轴/单轴东西/南北/极轴/垂直轴} = 15+ 个
- `SG_SZA/SG_SAA` 逐小时 24 个 × 多种统计量
- `EQUIV_NO_SUN_CONSEC` × {1/3/7/14/21天/月} 连续无太阳天数
- `SOLAR_DEFICITS/SURPLUS_INSOL_CONSEC` 太阳能盈余/赤字统计

**查询参数元数据示例**：

```python
import requests

# 查询 RE 社区 HOURLY 全部参数（含元数据）
url = ("https://power.larc.nasa.gov/api/system/manager/parameters?"
       "user=nasapower4r&community=RE&temporal=HOURLY&metadata=true")
params = requests.get(url, timeout=60).json()
# 返回 dict: {"T2M": {"name":"Temperature at 2 Meters","units":"C",
#               "definition":"...","type":"METEOROLOGY","source":"MERRA-2"}, ...}

# 查询单个参数详情
url = "https://power.larc.nasa.gov/api/system/manager/parameters/T2M?user=nasapower4r"
```

**参数字段结构**（每个参数含）：
`name`（长名）、`units`（单位）、`definition`（定义）、`type`（类型：METEOROLOGY/RADIATION/...）、`source`（数据源）、`units_min/max`（值域）、`data_type`（float32 等）、`calculated`（是否计算派生）

完整参数清单已归档至 `.knowledge/tech/nasa_power_params.md`。

#### 9. 速率限制

- 无 API key，但有隐式限速
- **并发不超过 5 个同步请求**（官方建议）
- HTTP 429 表示请求过多，可能被封禁
- 避免小于 0.5°（约 50km）的过密网格请求（会重复请求同一格点）

#### 10. 测试验证结果（2026-07-20）

**API 测试**：

| 测试项 | 结果 | 说明 |
|--------|------|------|
| 匿名访问 | ✅ 成功 | 无需注册、无需 key |
| Hourly API | ✅ 成功 | 240 条（10 天 × 24 小时） |
| Daily API | ✅ 成功 | 10 条日数据 |
| 辐射数据（2026-07） | ❌ 缺失（-999） | CERES 延迟超 19 天 |
| 辐射数据（2025-07） | ✅ 可用 | 延迟约 3-4 个月 |
| 气象数据（2026-07） | ✅ 可用 | MERRA-2/GEOS-IT 延迟约 2 天 |

**与 SURFRAD 实测对比**（Bondville 2025-07-01~10，240 小时）：

| 指标 | GHI | 温度（T2M） |
|------|-----|------------|
| SURFRAD 实测均值 | 319.0 W/m² | 25.0°C |
| NASA POWER 均值 | 295.6 W/m² | 25.8°C |
| **MAE** | **38.6 W/m²** | **1.4°C** |
| **RMSE** | **68.0 W/m²** | — |
| **BIAS** | **-23.3 W/m²**（轻度低估） | +0.8°C |

**精度分析**：
- GHI 系统性轻度低估白天峰值（约 5-8%），夜间两者均接近 0
- 温度 MAE 仅 1.4°C，精度优秀
- POWER 峰值 942 W/m² vs SURFRAD 1035 W/m²，低估约 9%

#### 11. 与 Open-Meteo（ERA5）对比

| 维度 | NASA POWER | Open-Meteo (ERA5) |
|------|-----------|-------------------|
| 辐射来源 | CERES 卫星反演 | ERA5 模型预报 |
| 气象来源 | MERRA-2 再分析 | ERA5 再分析 |
| 空间分辨率 | 0.5°-1°（50-100km） | 0.25°（31km） |
| 时间范围 | 1981 起 | 1940 起 |
| 小时数据起始 | 2001 年 | 1940 年 |
| 辐射延迟 | 3-4 个月 | 约 5 天 |
| 气象延迟 | 2 天 | 5 天 |
| API key | 不需要 | 不需要 |
| 速率限制 | 并发 ≤5 | 1 万次/天 |
| 输出格式 | CSV/JSON/NetCDF/EPW/ICASA | JSON/CSV |
| GHI 精度（vs SURFRAD） | MAE 38.6 W/m² | 待测 |

**互补价值**：POWER 辐射来自卫星观测（CERES），Open-Meteo 辐射来自模型预报（ERA5），两套独立数据源可交叉验证。当实测站点稀疏时，卫星反演可能优于纯模型。

### 禁止做法（avoid）

- ❌ 假设辐射数据实时可用（CERES 延迟 3-4 个月，需用历史数据或气象参数）
- ❌ 混淆 hourly（Wh/m²）和 daily（kW·h/m²/day）的单位
- ❌ 用于气候趋势分析（数据源拼接导致时间序列不连续）
- ❌ 用于城市微气候或复杂地形（0.5°-1° 分辨率太粗）
- ❌ 并发超过 5 个请求（会被封禁）
- ❌ 过密网格请求（<0.5° 间距无意义，同一格点重复）

## 适用场景

- 光伏/光热项目前期选址（GHI/DNI 估算）
- 农业作物模型（DSSAT/ICASA 兼容）
- 建筑能耗分析（EPW 文件输出）
- 无实测站点的偏远地区初步评估
- 与 ERA5 交叉验证辐射数据
- 长期历史辐射分析（1984 起，注意数据源拼接）

## 不适用场景

- 实时辐射监测（CERES 延迟 3-4 个月，应选 SURFRAD 实测）
- 高分辨率空间分析（0.5°-1° 太粗，应选 ERA5 的 0.25°）
- 城市微气候/风场微观选址（分辨率不足）
- 气候趋势归因（数据源拼接不连续）
- 1980s-1990s 的逐时分析（小时数据仅 2001 起）
